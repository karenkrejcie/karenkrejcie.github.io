Step 1: Create an Account on Github.com

Already have an account? Skip to Step 2.

Github is a cloud-based version control system as its core. There's a lot of functionality to Github, but the one we will use for this exercise is creating a public-facing website you can call your own!

To get started, you have to give some details about yourself to Github. Nothing is free, remember. Let's make it happen:

  1. Navigate to the Github Sign-Up page.
  2. Enter a valid email address you can access when prompted:
  3. Next, enter a super sleuthy password that meets all the rules of being sleuthy:
  4. Time to come up with a unique username. Nice try, but "karenkrejcie" is already taken.
  5. If you want to read through some product updates and occasional email from Github, now's your chance. Choose wisely.
  6. Now it is time to be a human. Do whatever it says to be human.
  7. Create your account!
  8. Login to your email used in #2 above, and retrieve the newly sent Launch Code
  9. Set up how your account will primarily be used.
  10. Install features as you see fit.
  11. Pick a version. Any version.

Step 2: Create a repo(sitory) for your site

Github will keep track of various projects you're working on in "repos" or repositories. You can name them anything you want NORMALLY. Since we are trying to make a web page on Github, we have to follow some Github protocols to have our free site. Here we go.

  1. Click on "Create a Repository" or "Start a Project":
  2. Name your Repository, but the name needs to be specific. Enter: username.github.io, substituting your username for username
  3. Create whatever HTML file you want to show up on your website's home page, and name the file index.html. Wow, that's bold. Must be important? You betcha. HOW do you do THAT, you ask? That's outside the scope of this documentation, but here's a snippet of some code you could write in a plain ol' text editor (Windows: Notepad; Mac: TextEdit with preference changes). Make sure you name it index.html and save it somewhere you can find it.
  4. Upload your existing index.html by clicking "Uploading an existing file":
  5. Drag your index.html into the "Drag files here" area in Github. Don't want to drag? Then browse. That's cool, too. Confirm index.html appears. Then, click "Commit changes" to make it official.

Step 3: Celebrate your new website

To see your website in all its glory, check out username.github.io substituting your username for username.
